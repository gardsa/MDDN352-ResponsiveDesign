Responsive Web Design

With such a large number of different-sized screens available, responsive web design is becoming more and more critical. There are a number of basic layout principles that we can utilise to embrace the fluidity of the web.

Responsive > Adapative

Responsive is fluid and adapts to the size of the screen, whereas adaptive uses static layouts at particular breakpoints.

Flow

Smaller screen sizes mean content takes up more vertical space, so you need to consider the ordering (or flow) of elements across all screen sizes.

Relative Units

Using units such as % and em over px means that your content will appear correctly over a variety of screen sizes.

Breakpoints

A breakpoint is a predefined point where the CSS rules change, like on this site where larger screens have 3 columns, but a mobile would only see 1 column.

Max/Min Values

Sometimes you want content to take up the full width of a screen, but on a larger screen this would look strange.

div { width: 100%; max-width: 1000px; }

Nested Objects

When a lot of elements depend on one another, things get complicated really fast. Wrapping groups of elements in separate containers keeps it clean and tidy.

Mobile or Desktop first?

Designing for mobile first allows you to progressively enhance your design. As mobile has the most limitations, you cater for these first, then add complexity for larger screens.

Fonts

Webfonts can result in a slick looking website, however each has to be downloaded, affecting overall page load and speed. System fonts are fast, but when not stored locally, will fall back to a default font.

Bitmap or Vectors?

Detailed images with fancy effects are better used in a bitmap format such as .jpg, .png, or .gif. Vectors are great for simple icons, with the best choice being either SVG or icon fonts.
